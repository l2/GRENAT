0007 `matlab2tikz` is a MATLAB(R) script to convert native MATLAB(R) figures to TikZ/Pgfplots figures that integrate seamlessly in LaTeX documents.
0008 
0009 To download the official releases and rate `matlab2tikz`, please visit its page on [FileExchange](http://www.mathworks.com/matlabcentral/fileexchange/22022).
0010 
0011 `matlab2tikz` converts most MATLAB(R) figures, including 2D and 3D plots. 
0012 For plots constructed with third-party packages, however, your mileage may vary.
0013 
0014 Installation
0015 ============
0016 
0017 Place the `matlab2tikz` functions (contents of `src/` folder) somewhere where MATLAB(R) can find them: either your working directory or add `src/` to the path using `addpath`.
0018 
0019 Make sure that your LaTeX installation is up-to-date and includes:
0020 
0021 * [TikZ/PGF](http://www.ctan.org/pkg/pgf) version 3.0 or higher
0022 * [Pgfplots](http://www.ctan.org/pkg/pgfplots) version 1.12.1 or higher
0023 * [Amsmath](https://www.ctan.org/pkg/amsmath) version 2.14 or higher
0024 * [Standalone](http://www.ctan.org/pkg/standalone) (optional)
0025 
0026 It is recommended to use the latest stable version of these packages.
0027 Older versions may work depending on the actual MATLAB(R) figure you are converting.

0029 Usage
0030 =====
0031 
0032 Typical usage of `matlab2tikz` consists of converting your MATLAB plot to a TikZ/LaTeX file and then running a LaTeX compiler to produce your document.
0033 
0034 MATLAB
0035 ------
0036   1. Generate your plot in MATLAB(R).
0037 
0038   2. Run `matlab2tikz`, e.g. using
0039 
0040 ```matlab
0041 matlab2tikz('myfile.tex');
0042 ```
0043 
0044 LaTeX
0045 -----
0046 Add the contents of `myfile.tex` into your LaTeX source code, for example using `\input{myfile.tex}`. 
0047 Make sure that the required packages (such as `pgfplots`) are loaded in the preamble of your document as in the example:
0048 
0049 ```latex
0050 \documentclass{article}
0051 
0052   \usepackage{pgfplots}
0053   \pgfplotsset{compat=newest}
0054   %% the following commands are sometimes needed
0055   \usetikzlibrary{plotmarks}
0056   \usepackage{grffile}
0057   \usepackage{amsmath}
0058   %% you may also want the following commands
0059   %\pgfplotsset{plot coordinates/math parser=false}
0060   %\newlength\figureheight
0061   %\newlength\figurewidth
0062 
0063 \begin{document}
0064   \input{myfile.tex}
0065 \end{document}
0066 ```

0068 Remarks
0069 -------
0070 Most functions accept numerous options; you can check them out by inspecting their help:
0071 
0072 ```matlab
0073 help matlab2tikz
0074 ```
0075 
0076 Sometimes, MATLAB(R) plots contain some features that impede conversion to LaTeX; e.g. points that are far outside of the actual bounding box.
0077 You can invoke the `cleanfigure` function to remove such unwanted entities before calling `matlab2tikz`:
0078 
0079 ```matlab
0080 cleanfigure;
0081 matlab2tikz('myfile.tex');
0082 ```
