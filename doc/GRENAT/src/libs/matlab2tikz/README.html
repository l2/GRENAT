0007 `matlab2tikz` is a MATLAB(R) script to convert native MATLAB(R) figures to TikZ/Pgfplots figures that integrate seamlessly in LaTeX documents.
0008 
0009 To download the official releases and rate `matlab2tikz`, please visit its page on [FileExchange](http://www.mathworks.com/matlabcentral/fileexchange/22022).
0010 
0011 `matlab2tikz` converts most MATLAB(R) figures, including 2D and 3D plots. 
0012 For plots constructed with third-party packages, however, your mileage may vary.
0013 
0014 Installation
0015 ============
0016 
0017 1. Extract the ZIP file (or clone the git repository) somewhere you can easily reach it. 
0018 2. Add the `src/` folder to your path in MATLAB/Octave: e.g. 
0019     - using the "Set Path" dialog in MATLAB, or 
0020     - by running the `addpath` function from your command window or `startup` script.
0021 
0022 Make sure that your LaTeX installation is up-to-date and includes:
0023 
0024 * [TikZ/PGF](http://www.ctan.org/pkg/pgf) version 3.0 or higher
0025 * [Pgfplots](http://www.ctan.org/pkg/pgfplots) version 1.13 or higher
0026 * [Amsmath](https://www.ctan.org/pkg/amsmath) version 2.14 or higher
0027 * [Standalone](http://www.ctan.org/pkg/standalone) (optional)
0028 
0029 It is recommended to use the latest stable version of these packages.
0030 Older versions may work depending on the actual MATLAB(R) figure you are converting.

0032 Usage
0033 =====
0034 
0035 Typical usage of `matlab2tikz` consists of converting your MATLAB plot to a TikZ/LaTeX file and then running a LaTeX compiler to produce your document.
0036 
0037 MATLAB
0038 ------
0039   1. Generate your plot in MATLAB(R).
0040 
0041   2. Run `matlab2tikz`, e.g. using
0042 
0043 ```matlab
0044 matlab2tikz('myfile.tex');
0045 ```
0046 
0047 LaTeX
0048 -----
0049 Add the contents of `myfile.tex` into your LaTeX source code, for example using `\input{myfile.tex}`. 
0050 Make sure that the required packages (such as `pgfplots`) are loaded in the preamble of your document as in the example:
0051 
0052 ```latex
0053 \documentclass{article}
0054 
0055   \usepackage{pgfplots}
0056   \pgfplotsset{compat=newest}
0057   %% the following commands are needed for some matlab2tikz features
0058   \usetikzlibrary{plotmarks}
0059   \usetikzlibrary{arrows.meta}
0060   \usepgfplotslibrary{patchplots}
0061   \usepackage{grffile}
0062   \usepackage{amsmath}
0063 
0064   %% you may also want the following commands
0065   %\pgfplotsset{plot coordinates/math parser=false}
0066   %\newlength\figureheight
0067   %\newlength\figurewidth
0068 
0069 \begin{document}
0070   \input{myfile.tex}
0071 \end{document}
0072 ```

0074 Remarks

0082 Sometimes, MATLAB(R) plots contain some features that impede conversion to LaTeX; e.g. points that are far outside of the actual bounding box.
0083 You can invoke the `cleanfigure` function to remove such unwanted entities before calling `matlab2tikz`:
0084 
0085 ```matlab
0086 cleanfigure;
0087 matlab2tikz('myfile.tex');
0088 ```
