0001 # Contributing to matlab2tikz
0002 
0003 You can contribute in many ways to `matlab2tikz`:
0004 
0005  - report bugs,
0006  - suggest new features,
0007  - write documentation,
0008  - fix some of our bugs and implement new features.
0009 
0010 The first part of this document is geared more towards users of `matlab2tikz`.
0011 The latter part is only relevant if you want to write some code for `matlab2tikz`.
0012 
0013 ## How to report a bug or ask for help
0014 
0015  1. Make sure you are using the [latest release](https://github.com/matlab2tikz/matlab2tikz/releases/latest) or even the [development version](https://github.com/matlab2tikz/matlab2tikz/tree/develop) of `matlab2tikz` and check that the problem still exists.
0016  2. Also make sure you are using a recent version of the required LaTeX packages (especially [`pgfplots`](http://ctan.org/pkg/pgfplots) and the [`TikZ`](http://ctan.org/pkg/pgf) libraries)
0017  3. You can submit your bug report or question to our [issue tracker](https://github.com/matlab2tikz/matlab2tikz/issues).
0018     Please, have a look at "[How to Ask Questions the Smart Way](http://www.catb.org/esr/faqs/smart-questions.html)" and "[Writing Better Bug Reports](http://martiancraft.com/blog/2014/07/good-bug-reports/)" for generic guidelines. In short:
0019     - Mention the version of MATLAB/Octave, the operating system, `matlab2tikz`, `pgfplots` and which `LaTeX` compiler you are using.
0020     - Choose a descriptive title for your issue report.
0021     - A short MATLAB code snippet that generates a plot where the problem occurs. Please limit this to what is strictly necessary to show the issue!
0022     - Explain what is wrong with the conversion of the figure (or what error messages you see).
0023     - Often it can be useful to also include a figure, `TikZ` code, ... to illustrate your point.
0024 
0025 ## How to request new features
0026 
0027 Please check first whether the feature hasn't been [requested](https://github.com/matlab2tikz/matlab2tikz/labels/feature%20request) before and do join the relevant topic in that case or maybe it has already been implemented in the [latest development version](https://github.com/matlab2tikz/matlab2tikz/tree/develop).
0028 
0029 If your feature is something new and graphical, please also have a look at the [`pgfplots`](https://www.ctan.org/pkg/pgfplots) manual to see if it supports the feature you want.
0030 In some cases it is more constructive to request the feature in the [`pgfplots` bug tracker](https://sourceforge.net/p/pgfplots/bugs/).
0031 
0032 Please submit you feature request as any [bug report](https://github.com/matlab2tikz/matlab2tikz/labels/feature%20request) and make sure that you include enough details in your post, e.g.:
0033 
0034  - What are you trying to do?
0035  - What should it look like or how should it work?
0036  - Is there a relevant section in the `pgfplots` or `MATLAB` documentation?
0037 
0038 ## Submitting pull requests (PRs)
0039 Before you start working on a bug or new feature, you might want to check that nobody else has been assigned to the relevant issue report.
0040 To avoid wasted hours, please just indicate your interest to tackle the issue.
0041 
0042 ### Recommended workflow
0043 [Our wiki](https://github.com/matlab2tikz/matlab2tikz/wiki/Recommended-git-workflow) contains more elaborate details on this process. Here is the gist:
0044 
0045  - It is highly recommended to start a feature branch for your work.
0046  - Once you have finished the work, please try to run the test suite and report on the outcome in your PR (see below).
0047  - Make sure that you file your pull request against the `develop` branch and *not* the `master` branch!
0048  - Once you have filed your PR, the review process starts. Everybody is free to join this discussion.
0049  - At least one other developer will review the code and signal their approval (often using a thumbs-up, :+1:) before the PR gets pulled into `develop`.
0050  - Once you have addressed all comments, one of the developers will merge your code into the `develop` branch.
0051 
0052 If you still feel uncomfortable with `git`, please have a look at [this page](https://github.com/matlab2tikz/matlab2tikz/wiki/Learning-git) for a quick start.
0053 
0054 ### Running the test suite
0055 We know that at first the test suite can seem a bit intimidating, so we tend to be lenient during your first few PRs. However, we encourage you to run the test suite on your local computer and report on the results in your PR if any failures pop up.
0056 To run the test suite, please consult its [README](https://github.com/matlab2tikz/matlab2tikz/blob/develop/test/README.md).
0057 
0058 ## Becoming a member of [matlab2tikz](https://github.com/matlab2tikz)
0059 
0060 Once you have submitted your first pull request that is of reasonable quality, you may get invited to join the [Associate Developers](https://github.com/orgs/matlab2tikz/teams/associate-developers) group.
0061 This group comes with *no* responsibility whatsoever and merely serves to make it easier for you to "claim" the features you want to work on.
0062 
0063 Once you have gained some experience (with `git`/GitHub, our codebase, ...) and have contributed your fair share of great material, you will get invited to join the [Developers](https://github.com/orgs/matlab2tikz/teams/developers) team.
0064 This status gives you push access to our repository and hence comes with the responsibility to not abuse your push access.
0065 
0066 If you feel you should have gotten an invite for a team, feel free to contact one of the [owners](https://github.com/orgs/matlab2tikz/teams/owners).
